# Тестовые образцы для hotword «Ким»

Эта директория содержит аудио-файлы для тестирования точности детекции hotword «Ким».

## Правила именования файлов

### Положительные примеры (должны срабатывать hotword)

Файлы, которые **должны** активировать hotword, должны содержать в имени:
- `positive` или `kim`

**Примеры:**
- `positive_kim_1.wav`
- `kim_activation.wav`
- `test_positive_1.mp3`
- `ким_активация.wav`

### Отрицательные примеры (не должны срабатывать hotword)

Файлы, которые **не должны** активировать hotword, должны содержать в имени:
- `negative` или `no_kim` или `no-kim`

**Примеры:**
- `negative_no_kim_1.wav`
- `no_kim_test.wav`
- `test_negative_1.mp3`
- `background_noise.wav`

## Поддерживаемые форматы

- `.wav` (рекомендуется)
- `.mp3`
- `.flac`
- `.ogg`
- `.m4a`

## Рекомендации по подготовке образцов

1. **Положительные примеры:**
   - Разные интонации произнесения «Ким»
   - Разная громкость
   - С фоновым шумом и без
   - Разные голоса (если возможно)
   - Разная скорость речи

2. **Отрицательные примеры:**
   - Похожие слова («кит», «кино», «кима»)
   - Фоновый шум без речи
   - Другие команды («Алиса», «Окей Гугл»)
   - Музыка
   - Тишина

## Минимальный набор для тестирования

Для базового тестирования рекомендуется иметь:
- **5-10 положительных примеров** (разные условия)
- **5-10 отрицательных примеров** (разные похожие слова и шумы)

## Запуск тестирования

```bash
python scripts/test_hotword_accuracy.py
```

Скрипт автоматически:
- Найдёт все аудио-файлы в этой директории
- Определит ожидаемое поведение по имени файла
- Протестирует каждый файл
- Вычислит метрики (Precision, Recall, F1-Score, Accuracy)
- Настроит параметры для достижения precision ≥ 0.9
- Сохранит отчёт в `test_hotword_accuracy.log`
